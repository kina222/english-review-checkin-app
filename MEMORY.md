# Memory

- 本项目是自用英语复习打卡 App，不再沿用旧的链上监控脚本方向。
- 第一版严格只做 MVP，不要为了好看或完整性增加复杂功能。
- 每条学习内容只保留 `english` 和 `chinese` 两个用户输入字段。
- 第一版不要做标签、来源、备注、登录、云同步、AI 纠错、朗读、OCR、社交、排行榜。
- 语音输入要同时保留手机系统输入法语音和 App 内单独语音按钮，因为系统语音有时需要跳转，不够稳定。
- App 内单独语音按钮允许浏览器麦克风授权弹窗；如果浏览器不支持或授权失败，就退回到普通输入框和系统输入法语音。
- 每天最多复习 15 条；超过 15 条的到期内容保持逾期状态，不能直接跳过。
- 完成当天全部被选中的复习任务后才算打卡成功，不能只打开页面就算打卡。
- “忘了 / 模糊 / 熟了”会影响下一次复习时间：新内容录入后第 2 天第一次复习；忘了 1 天后，模糊 3 天后，熟了按 3 天后、第 7 天、第 15 天、第 30 天、之后每 30 天的正常下一档。
- 当前 Web 版已加入 `JSON` 合并导入；导入时跳过英文和中文都重复的现有内容，只补充新内容，不能覆盖或清空用户当前数据。
- 手动新增也必须拦截重复：英文忽略大小写和多余空格、中文忽略多余空格，英文和中文都重复时提示已在内容库，不新增第二条。
- 当前 Web 版已加入英文朗读按钮；朗读使用浏览器/系统 `speechSynthesis`，不存储音频文件，不改变每条内容仍只有 `english` 和 `chinese` 两个用户输入字段。
- 当前 Web 版已加入轻量音效，使用浏览器 `Web Audio` 生成，不存储音频文件；复习发音提示使用本地常见词 IPA 词表，属于辅助提示，不要当作完整权威词典。
- iPhone 输入框聚焦自动放大已做 Web 侧优化：输入框保持 16px，viewport 设置 `maximum-scale=1, user-scalable=no, viewport-fit=cover`，并设置 `text-size-adjust: 100%`；若仍跳动，多半是 iOS WebView 限制，需要继续从布局稳定性处理。
- 复习页支持加练：完成当前组后可以“再来一组”，每组 15 条，每天最多 5 组；不要为了加练清掉当天已完成打卡，避免用户加练中断导致连续打卡丢失。
- 复习页支持手写/口语模式切换；口语模式只把语音识别文字写入当前答案框，不改变词条数据结构。
- “忘了”不能立即跳下一条；第一次点击先停留显示标准答案和朗读按钮，用户确认“看完了，记为忘了”后再记录结果和跳下一条。
- 正式版可以借鉴 Gemini 预览的黄色顶栏、底部导航、复习卡片和航海主题文案，但不能直接套用它的导入、危险清空、外部 Tailwind 依赖或不一致复习规则。
- 轻量娜美航海主题可以保留在文案和颜色里，但功能字段、存储结构和复习规则必须以本项目文档为准。
- 遇到需求不清楚时，先写入 `docs/requirements.md` 的“待确认问题”，不要自行扩大范围。
- 每次有效变更必须写入 `CHANGELOG.md`。
- 线上版本更新不能更换 `localStorage` 主数据 key；如果以后必须调整存储结构，要先做迁移和导出备份提示，避免用户已录入内容丢失。
- 用户已经开始连续打卡；后续版本更新必须保护 `dailyCheckIns` 和 `STORAGE_KEY`，不能因为界面或功能更新让打卡记录中断。
- 用户数据按网址隔离；`localhost`、旧 GitHub Pages 地址、当前 `kina222.github.io/english-review-checkin-app` 和手机主屏幕入口可能不是同一份本地数据。正式使用后固定当前线上地址。
