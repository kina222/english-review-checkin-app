# 英语复习打卡 App

这是一个自用英语复习打卡工具，目标是在手机上像 App 一样一键打开，用最少操作记录和复习最近学到的英语内容。

第一版只做 MVP：先让记录、复习、打卡和备份跑通，不追求复杂功能和漂亮页面。

## 这个 App 用来做什么

你可以用它记录最近学到的英语内容，例如：

- 单词
- 短语
- 短句
- 完整句子
- 动画台词
- 宝宝英语启蒙内容
- 日常亲子对话

每条内容只需要填写两个字段：

- `english`：英文
- `chinese`：中文

## 复习规则

新增一条内容后，基础复习节奏是：

- 第 3 天
- 第 7 天
- 第 15 天
- 第 30 天
- 之后每 30 天

每天最多复习 15 条。

如果当天到期内容超过 15 条，系统只选出 15 条进入当天复习。剩余内容保持逾期状态，之后仍然会继续进入待复习池，不会被跳过或删除。

## 每日复习怎么做

每天进入复习时：

1. 系统选出今天到期和逾期内容，最多 15 条
2. 每条内容随机显示英文或中文
3. 答案默认遮挡
4. 点开后显示答案
5. 选择“忘了 / 模糊 / 熟了”
6. 完成全部被选中的内容后，才算当天打卡成功
7. 最后显示每日总结

## 掌握反馈怎么影响下次复习

- 忘了：1 天后再出现
- 模糊：3 天后再出现
- 熟了：按正常节奏进入下一档
- 第 30 天之后如果选择熟了：继续 30 天后再出现

## 语音输入怎么处理

第一版保留两种语音输入：

- 手机系统输入法自带语音输入
- App 内单独语音输入按钮

App 内语音按钮允许浏览器弹出麦克风授权。如果当前浏览器不支持，或者授权失败，就退回到普通输入框，继续使用手机系统输入法语音。

以后在手机上使用 App 内语音按钮时，也需要浏览器或系统允许麦克风权限。添加到主屏幕后，本质仍然是 Web App，所以第一次使用语音时通常还是会弹出权限确认。

## 数据保存和备份

第一版数据保存在本地，不需要登录，也不做云同步。

第一版必须支持一键导出 `JSON` 备份。导出的文件用于防止本地数据丢失，比如浏览器清缓存、换手机或后续升级版本。

第一版暂时不做导入备份。导入属于以后恢复数据用的功能，等 MVP 跑通后再考虑。

## 启动方式

运行：

```bash
npm run dev
```

启动后在浏览器打开终端显示的本地地址。

当前项目没有外部依赖，不需要先运行 `npm install`。

## 在线部署

推荐先部署为在线 PWA。

部署目标：

- GitHub 仓库：`kina222/english-review-checkin-app`
- 部署方式：GitHub Pages 或其他静态网站服务

上线后，手机打开线上网址，再添加到主屏幕。这样电脑关机、换 Wi-Fi、使用手机流量时也能打开。

注意：数据仍然保存在手机浏览器本地。换手机或清理浏览器数据前，要先导出 `JSON` 备份。

## 界面方向

当前界面采用手机优先的航海手账风格：
