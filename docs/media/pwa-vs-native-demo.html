PWA 和手机 App 封装区别

给英语复习打卡 App 的路线选择

开发测试版
💻

手机访问电脑开的本地服务。

不适合日常使用

电脑一关，手机就打不开

  • 需要电脑一直开着 `npm run dev`
  • 手机和电脑必须在同一个 Wi-Fi
  • 换网络、出门、电脑睡眠都会失败
在线 PWA
🌐

手机打开一个固定网址，然后添加到主屏幕。

推荐先做

电脑关了也能用

  • App 文件放到 GitHub Pages / Cloudflare Pages
  • 手机换 Wi-Fi、用流量也能打开
  • 数据仍然保存在你的手机浏览器本地
封装成手机 App
📦

把网页包进原生 App 壳里。

以后再考虑

更像 App，但成本更高

  • 需要额外工具，例如 Capacitor
  • iPhone 会遇到证书、签名、安装问题
  • 每次更新也比网页部署麻烦
建议路线
🍊

先做在线 PWA，稳定用起来。

最适合当前 MVP

先上线一个网址，再加到手机桌面

  • 不用电脑常开
  • 不需要应用商店
  • 后续真有必要，再封装成原生 App

结论：现在先做在线 PWA。它最轻、最稳、最适合自用 MVP。
