# 排障手册

## 新增内容后没有进入复习

可能原因：

- 新内容不是当天立即复习，而是按第 3 天开始安排
- 本地日期不正确
- 保存时 `nextReviewAt` 没有正确写入

处理方式：

- 先检查内容库里的下一次复习日期
- 再检查手机或电脑系统日期
- 最后检查本地存储中的 `nextReviewAt`

## 今天到期内容超过 15 条

可能原因：

- 累积内容较多
- 多天没有复习

处理方式：

- 第一版每天最多只显示 15 条
- 超出的到期内容保持逾期状态
- 逾期内容后续仍然进入待复习池
- 不要把超出的内容标记为已完成或跳过

## 打开页面但没有打卡成功

原因：

- 第一版规则是完成当天全部被选中的复习任务后才算打卡成功
- 只打开页面或只完成部分复习都不算打卡

处理方式：

- 完成每日复习列表中的全部条目
- 在每日总结中确认打卡状态

## 语音输入不可用

可能原因：

- 手机系统输入法语音需要跳转或权限不稳定
- App 内语音按钮依赖浏览器支持
- 浏览器或输入法没有开启麦克风权限
- 手机添加到主屏幕后，本质仍然是 Web App，App 内语音仍然需要浏览器或系统授权麦克风
- 当前设备键盘没有语音按钮

处理方式：

- 优先尝试 App 内语音按钮
- 如果 App 内语音按钮不可用，退回到系统输入法语音
- 检查浏览器麦克风权限
- 手机上第一次使用 App 内语音时，看到麦克风权限弹窗要选择允许
- 检查输入法设置

## 数据丢失风险

可能原因：

- 浏览器本地数据被清理
- 更换设备
- 使用无痕模式
- 浏览器存储被系统回收

处理方式：

- 定期使用一键导出 `JSON` 备份
- 不要在无痕模式长期使用
- 重要阶段先导出备份再做改动

## 导出备份失败

可能原因：

- 浏览器阻止下载
- 本地存储数据异常
- 手机系统下载权限受限

处理方式：

- 先换浏览器尝试
- 再检查浏览器下载权限
- 如果是数据异常，先保留当前页面，不要清理浏览器数据

## `npm install` 长时间没有输出

可能原因：

- 当前网络访问依赖源不稳定
- 第一版其实不需要外部依赖

处理方式：

- 不需要运行 `npm install`
- 直接使用 `npm run dev` 启动
- 如果后续引入外部依赖，再重新补充安装步骤

## 页面打开后是空白

可能原因：

- 浏览器脚本加载失败
- 把需要打包工具处理的写法直接放到了零依赖静态服务器里

处理方式：

- 先刷新页面
- 如果仍然空白，检查浏览器开发者工具里的 Console 报错
- 零依赖模式下，样式文件要通过 `index.html` 的 `<link>` 引入，不能在 `main.js` 里直接 `import` CSS
