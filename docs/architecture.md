# 技术架构

## 架构目标

- 快速做出可用 MVP
- 手机上像 App 一样一键打开
- 本地保存数据
- 支持一键导出备份
- 结构简单，方便后续维护

## 推荐方向

第一版优先做轻量 Web App。

原因：

- 手机浏览器可以直接打开
- 可以添加到手机主屏幕
- 不需要上架应用商店
- 开发和修改成本低
- 适合自用 MVP

## 模块划分

### 输入模块

负责新增英语内容：

- 手动输入
- 复制粘贴
- 手机系统键盘语音输入
- App 内单独语音输入按钮

App 内单独语音输入按钮优先使用浏览器可用的语音识别能力，允许出现麦克风授权弹窗。如果当前浏览器不支持或授权失败，则退回到普通输入框和系统输入法语音。

### 内容库模块

负责展示已保存内容：

- 英文
- 中文
- 下一次复习日期

第一版不做复杂筛选和分类。

### 复习计划模块

负责计算下一次复习时间：

- 第 3 天
- 第 7 天
- 第 15 天
- 第 30 天
- 之后每 30 天

反馈会影响下一次复习时间，具体规则以 `docs/requirements.md` 为准。

### 每日复习模块

负责当天复习流程：

- 选出今天到期和逾期内容
- 限制最多 15 条
- 随机决定先显示英文或中文
- 遮挡答案
- 记录“忘了 / 模糊 / 熟了”
- 根据反馈更新下一次复习时间
- 全部完成后生成打卡结果

### 本地存储模块

负责保存：

- 学习内容
- 复习记录
- 每日打卡状态

第一版优先考虑浏览器本地存储能力。

### 备份模块

负责一键导出 `JSON`：

- 导出全部学习内容
- 导出全部复习记录
- 导出全部每日打卡记录
- 文件名包含导出日期

## 手机桌面入口

第一版需要支持添加到手机主屏幕。

基础要求：

- 有页面标题
- 有应用图标
- 有基础配置文件
- 手机浏览器打开后可以添加到主屏幕

当前实现：

- 使用 `manifest.webmanifest` 提供应用名称、图标和启动方式
- 使用 `service-worker.js` 缓存基础文件
- 使用相对资源路径，兼容 GitHub Pages 子路径部署
- 数据仍保存在手机浏览器本地

## 暂不建设的能力

- 登录系统
- 云同步服务
- 后端服务
- AI 服务
- OCR 服务
- 社交系统
- 排行榜
- 复杂统计看板
