# 页面流程

## 设计原则

- 手机优先
- 打开就能用
- 不做复杂前端 UI
- 不用过多说明文字占据主流程
- 优先减少输入和复习阻力

## 主要页面

### 首页

首页承担当天入口：

- 显示今天是否已打卡
- 显示今天待复习数量
- 显示逾期待复习数量
- 提供进入每日复习的入口
- 提供新增内容入口
- 提供内容库入口
- 提供导出备份入口

### 新增内容

新增内容页面只包含核心输入：

- `english` 输入框
- `chinese` 输入框
- 保存按钮

输入方式：

- 手动输入
- 复制粘贴
- 手机系统键盘语音输入
- App 内单独语音输入按钮

App 内单独语音输入按钮用于减少系统输入法跳转带来的不稳定体验。

保存后：

- 创建内容记录
- 生成复习计划
- 返回首页或继续新增

### 内容库

内容库用于查看已保存内容：

- 显示英文
- 显示中文
- 显示下一次复习日期
- 支持基础查看

第一版不做标签、来源、备注和复杂筛选。

### 每日复习

每日复习流程：

1. 系统选出今天到期和逾期内容，最多 15 条
2. 每条内容随机显示英文或中文
3. 另一侧答案默认遮挡
4. 用户点击后显示答案
5. 用户选择“忘了 / 模糊 / 熟了”
6. 根据反馈更新下一次复习时间
7. 进入下一条
8. 全部完成后进入每日总结

超过 15 条的内容继续保持逾期状态，后续仍然会进入待复习池。

### 每日总结

每日总结显示：

- 今天完成复习条数
- “忘了 / 模糊 / 熟了”数量
- 是否打卡成功

只有完成当天全部被选中的复习任务后，才算打卡成功。

### 导出备份

导出备份入口放在首页或设置区域。

第一版能力：

- 一键导出本地全部数据
- 文件格式为 `JSON`
- 文件名建议包含导出日期

## 第一版不出现的入口

- 登录入口
- 云同步入口
- 社交分享入口
- 排行榜入口
- AI 纠错入口
- 朗读按钮
- OCR 入口
- 标签管理入口
